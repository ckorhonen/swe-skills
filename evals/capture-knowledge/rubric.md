# `swe:capture-knowledge` Rubric

## Pass Conditions

- Compares existing guidance to observed repo patterns before proposing gaps.
- Keeps proposed entries tightly scoped to missing, behavior-changing guidance.
- Includes evidence for each proposed guidance entry.
- Stops at draft or review-packet output when the case does not authorize
  immediate write-back.

## Fail Conditions

- Summarizes the repo without comparing it to existing guidance.
- Repeats guidance that is already documented well enough.
- Proposes trivia that would not change future agent behavior.
- Writes or recommends write-back without an explicit review step when the case
  is draft-only.

## Common Failure Modes

- Treating general onboarding notes as reusable agent guidance.
- Missing the difference between repo reality and documented rules.
- Skipping evidence and presenting speculative conventions as fact.
- Jumping straight to file edits instead of producing a review packet.
